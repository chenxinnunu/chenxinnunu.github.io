意思就是说SELECT后面要查询的字段telephone不在GROUP BY子句中并且包含非聚集列’telephone’，也查了相关资料，原来是 MySQL 5.7.5及以上加了功能依赖检测功能。默认情况下启用了ONLY_FULL_GROUP_BY SQL模式，MySQL将拒绝选择列表（5.7.5之前，MySQL没有检测到功能依赖关系，默认情况下不启用ONLY_FULL_GROUP_BY。

但是为什么根据id分组不报错呢？因为用户的id是唯一主键，查出的telephone就是唯一的，直接显示就好了。而根据name查出来的telephone可能有多个，MySQL拒绝从多个列中取值。
